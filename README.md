Queue Bot для Telegram

Queue Bot — это небольшой помощник для групп, где люди постоянно ждут свою очередь: созвоны, код-ревью, запись на консультацию, приём задач и т.д.  
Проект делался «для жизни»: чтобы можно было запустить, выдать людям кнопки и не объяснять полчаса, как всё работает.

Идея простая:  
люди не должны тонуть в сообщениях “кто последний?”, а админ — вручную следить за порядком.


Небольшой сюжет 

Представь рабочий чат, где каждый день десятки заявок.  
Пока всё вручную — кто-то теряет место, кто-то забывает ответить, кто-то пишет “я вроде после Иры”.

Queue Bot приходит как «тихий диспетчер»:  
- фиксирует порядок;
- напоминает, когда очередь подошла;
- двигает очередь дальше, если человек пропал;
- даёт людям работать через личку, не листая групповой флуд.

В итоге у команды меньше хаоса, а у админа меньше головной боли.


Что умеет:

Для обычных участников
- Встать в очередь / выйти из очереди.
- Посмотреть свои очереди в личке (`/myqueues`) и в группе (`/myplace`).
- Получить DM, когда очередь подошла.
- Поставить персональный ник для конкретной группы (`/me`).
- Подписаться на уведомление, если в ограниченной очереди освободится место.

Проще говоря: участник заходит в личку, нажимает 1–2 кнопки и понимает, где он стоит и что делать дальше.

Для админов
- Создавать очереди с названием, описанием и лимитом мест.
- Закрывать/удалять очереди.
- Кикать участников вручную.
- Настраивать:
  - таймаут напоминания,
  - публичное уведомление о выходе,
  - авто-кик за неактивность.
- Делать CSV-экспорт очереди.

То есть админ управляет процессом, а не вручную переносит людей в сообщениях.

Для владельца бота
- Назначать ботовых админов (`/addadmin`, `/removeadmin`, `/admins`).
- Делать рассылки (`/broadcast`).


Стек:
- Python 3.11+
- [aiogram 3.x](https://docs.aiogram.dev/)
- SQLite (по умолчанию) или PostgreSQL
- FastAPI + Uvicorn (опциональная web-панель)
